1. Descripción General

eCertify es una plataforma web desarrollada en ASP.NET Core 8 (Razor Pages) que automatiza el proceso de certificación como Emisor de Comprobantes Fiscales Electrónicos (e-CF) ante la Dirección General de Impuestos Internos (DGII) de la República Dominicana.

La plataforma guía a cada empresa a través de los pasos del proceso de certificación: registro, requisitos, postulación, pruebas técnicas, aprobación comercial, simulación, representación gráfica, URL de servicios, declaración jurada y firma digital de documentos XML.

Objetivo principal: Reducir el tiempo y la complejidad técnica del proceso de certificación e-CF, permitiendo a empresas de cualquier tamaño cumplir con la normativa DGII de forma guiada y automatizada.

4. Sistema de Autenticación

4.1 Flujo JWT

Al iniciar sesión, el servidor genera un token JWT (HS256) con duración de 120 minutos. El token se almacena en una cookie HttpOnly y el HttpClient "ApiClient" lo inyecta automáticamente en cada petición interna.

4.2 Claims del Token

ClaimDescripción
RNCRNC de la empresa activa del usuario
EmpresaIDID numérico de la empresa en base de datos
PlanIDPlan de suscripción — controla acceso a funcionalidades
EmailCorreo del usuario autenticado

4.3 Validación de Plan

El servicio PlanValidator verifica en cada OnGetAsync() que el usuario tenga un plan de suscripción activo. Si no, redirige a la página de selección de plan.

5. Sistema de Firma Digital XML

5.1 Estándar Implementado

La firma cumple el estándar W3C XMLDSIG (XML Digital Signature Syntax and Processing). Utiliza el certificado X.509 (.p12) de la empresa almacenado localmente.

  • Algoritmo de firma: http://www.w3.org/2001/04/xmldsig-more#rsa-sha256
  • Algoritmo de digest: http://www.w3.org/2001/04/xmlenc#sha256
  • Transformación: XmlDsigEnvelopedSignatureTransform

5.2 Flujo Completo de Firma

  1. El usuario sube un archivo XML en /Certificacion/FirmarXml.
  2. La página envía POST api/FirmarXml/FirmarXml con el archivo y el RNC.
  3. FirmarXmlController busca la empresa por RNC en la base de datos, obtiene nombre y contraseña del certificado.
  4. SemillaService.FirmarXml() carga el X509Certificate2 desde Storage/Certificados/{RNC}/.
  5. Extrae la clave privada RSA, configura SignedXml, calcula y añade el elemento <Signature> al documento XML.
  6. El controlador retorna el XML firmado como descarga (application/xml).

5.3 Página Unificada FirmarXml

La ruta /Certificacion/FirmarXml reemplaza las antiguas páginas Postulacion, DeclaracionJurada y Firmar. Acepta el parámetro ctx para adaptar su comportamiento:

ctxContextoPaso Rastreado
postulacionPostulación e-CFPaso 1
declaracionDeclaración JuradaPaso 9
(vacío)Firma libre genéricaNinguno

11. Guía de Despliegue

11.1 Requisitos

  • .NET 8 Runtime instalado en el servidor
  • SQL Server (Express o superior)
  • Acceso a internet (endpoints DGII)
  • Certificados .p12 de cada empresa en Storage/Certificados/{RNC}/

11.2 Pasos

  1. Clonar o copiar el proyecto al servidor.
  2. Configurar appsettings.json con la cadena de conexión y claves JWT.
  3. Ejecutar migraciones: dotnet ef database update
  4. Publicar: dotnet publish -c Release -o ./publish
  5. Configurar IIS o servicio systemd apuntando al ejecutable publicado.
  6. Colocar los certificados .p12 en Storage/Certificados/{RNC}/.
